# ⚔️ ChargebackDefender

AI-powered chargeback dispute letter generator for Shopify, Etsy, Amazon, PayPal, and Stripe sellers.
Win rate: 60–80% vs. ~20% DIY. $9/dispute or unlimited via Nas.io community.

---

## Stack

| Layer      | Tech                               |
|------------|------------------------------------|
| Backend    | FastAPI + Uvicorn + SQLAlchemy     |
| Database   | PostgreSQL (async via asyncpg)     |
| AI         | Anthropic Claude (claude-sonnet)   |
| Auth       | JWT (python-jose + passlib/bcrypt) |
| Frontend   | React 18 + Vite + Zustand          |
| Payments   | Nas.io webhooks + Stripe credits   |
| Deploy     | Render Blueprint                   |

## Local Development

### 1. Clone & setup

```bash
git clone https://github.com/yourname/chargeback-defender.git
cd chargeback-defender
```

### 2. Backend setup

```bash
cd backend

# Create virtual environment
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Configure environment
cp .env.example .env
# → Edit .env with your ANTHROPIC_API_KEY and DATABASE_URL
```

### 3. Database setup (local PostgreSQL)

```bash
# Option A: Docker (recommended)
docker run -d \
  --name chargeback-db \
  -e POSTGRES_DB=chargebackdb \
  -e POSTGRES_USER=user \
  -e POSTGRES_PASSWORD=password \
  -p 5432:5432 \
  postgres:16-alpine

# Option B: Local PostgreSQL
createdb chargebackdb
```

Update your `backend/.env`:
```
DATABASE_URL=postgresql://user:password@localhost:5432/chargebackdb
```

### 4. Run backend

```bash
# From /backend directory
uvicorn main:app --reload --port 8000

# API docs available at:
# http://localhost:8000/api/docs  (DEBUG=true only)
# http://localhost:8000/api/health
```

### 5. Frontend setup

```bash
cd ../frontend

# Install dependencies
npm install

# Dev server (proxies /api → localhost:8000)
npm run dev
# → http://localhost:5173
```

## Render Deployment (Git-to-Render)

### Step 1: Push to GitHub

```bash
cd chargeback-defender

git init
git add .
git commit -m "feat: initial ChargebackDefender boilerplate"

# Create repo on GitHub, then:
git remote add origin https://github.com/YOUR_USERNAME/chargeback-defender.git
git branch -M main
git push -u origin main
```

### Step 2: Connect Render Blueprint

```bash
# Option A: Via CLI
npm install -g @render-com/cli      # or: brew install render
render login
render blueprint sync               # Reads render.yaml and creates all services

# Option B: Via Dashboard
# 1. Go to https://dashboard.render.com
# 2. Click "New" → "Blueprint"
# 3. Connect your GitHub repo
# 4. Render auto-reads render.yaml and creates:
#    - chargeback-api (Web Service)
#    - chargeback-frontend (Static Site)
#    - chargeback-db (PostgreSQL)
```

### Step 3: Set secret environment variables

After deploy, go to each service in Render Dashboard → Environment:

**chargeback-api → Environment Variables:**
```
ANTHROPIC_API_KEY        = sk-ant-...
NASIO_WEBHOOK_SECRET     = [from Nas.io dashboard]
STRIPE_SECRET_KEY        = sk_live_...
STRIPE_WEBHOOK_SECRET    = whsec_...
```

> `JWT_SECRET_KEY` is auto-generated by Render (set `generateValue: true` in render.yaml)

### Step 4: Update frontend API URL

In Render Dashboard → chargeback-frontend → Environment:
```
VITE_API_URL = https://chargeback-api.onrender.com/api
```

Then trigger a redeploy:
```bash
render deploy --service chargeback-frontend
```

### Step 5: Verify deployment

```bash
# Health check
curl https://chargeback-api.onrender.com/api/health
# → {"status": "ok", "service": "ChargebackDefender"}

# Test registration
curl -X POST https://chargeback-api.onrender.com/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"testpass123"}'
```

## Nas.io Integration Setup

### 1. Configure webhook in Nas.io dashboard

```
Webhook URL:  https://chargeback-api.onrender.com/api/webhooks/nasio
Events:       member.joined, member.left, member.renewed, member.payment_failed
Secret:       [copy to NASIO_WEBHOOK_SECRET in Render env]
```

### 2. Configure Magic Link redirect

In Nas.io → Community Settings → Integrations → Magic Link:
```
Redirect URL: https://chargeback-api.onrender.com/api/webhooks/nasio/magic-link
```

### 3. How it works

```
Buyer joins Nas.io community
  → Nas.io sends member.joined webhook
  → ChargebackDefender finds/creates user by email
  → Sets is_premium = true for 32 days
  → Buyer uses Magic Link → auto-logged in with premium access
```

## Pay-per-Dispute ($9/credit) via Stripe

### 1. Create Stripe Payment Link

In Stripe Dashboard → Payment Links → Create:
```
Product:          "1 Dispute Credit"
Price:            $9.00 one-time
Metadata:         dispute_credits = 1
Customer email:   Collect customer email ✓
```

### 2. Set Stripe webhook

```
Stripe Dashboard → Webhooks → Add endpoint:
URL:    https://chargeback-api.onrender.com/api/webhooks/stripe
Events: checkout.session.completed
```

## API Reference

| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| POST | /api/auth/register | — | Create account |
| POST | /api/auth/login | — | Get JWT |
| GET | /api/auth/me | JWT | Profile + credits |
| GET | /api/disputes/ | JWT | List disputes |
| POST | /api/disputes/ | Premium | Create dispute |
| POST | /api/disputes/{id}/evidence | Premium | Upload + OCR files |
| POST | /api/disputes/{id}/generate | Premium | Trigger AI generation |
| GET | /api/disputes/{id} | JWT | Get dispute + letter |
| GET | /api/disputes/{id}/download | JWT | Download letter |
| GET | /api/disputes/reason-codes | — | List supported codes |
| POST | /api/webhooks/nasio | HMAC | Nas.io events |
| GET | /api/webhooks/nasio/magic-link | — | Magic link handler |
| POST | /api/webhooks/stripe | HMAC | Stripe events |
